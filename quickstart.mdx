## Development Setup

Learn how to set up your bot locally and deploy it to production.

### Installation and Local Development

<AccordionGroup>
  <Accordion icon="github" title="Clone the repository">
    Get started by cloning the Bleed Bot repository to your local machine:
    ```bash
    git clone https://github.com/your-username/bleed-bot-enhanced.git
    cd bleed-bot-enhanced
    ```

    The repository contains all the necessary files for running your enhanced Discord bot.
  </Accordion>
  
  <Accordion icon="rectangle-terminal" title="Install dependencies">
    Before running the bot, you'll need to install all required dependencies:
    ```bash
    npm install
    ```
    
    This will install:
    - discord.js (Discord API library)
    - mongoose (MongoDB integration)
    - express (API server)
    - and other essential packages
  </Accordion>
  
  <Accordion icon="robot" title="Configure your bot">
    1. Copy the example config file:
    ```bash
    cp config/config.example.json config/config.json
    ```
    
    2. Edit `config/config.json` with your:
    - Discord bot token
    - Client ID
    - Test server ID
    - Owner ID(s)
    
    3. For database functionality, configure MongoDB URI in the config file
  </Accordion>
  
  <Accordion icon="rocket" title="Run the bot">
    Start the bot in development mode (with auto-restart):
    ```bash
    npm run dev
    ```
    
    Or for production:
    ```bash
    npm start
    ```
  </Accordion>
</AccordionGroup>

### Deployment

<AccordionGroup>
  <Accordion icon="server" title="Hosting options">
    You can host your Bleed Bot on:
    - VPS (DigitalOcean, Linode, AWS EC2)
    - Container services (Docker, Kubernetes)
    - Serverless platforms (with adjustments)
    - Specialized bot hosting (PebbleHost, etc.)
    
    <Note>
      For 24/7 uptime, consider using process managers like PM2:
      ```bash
      npm install -g pm2
      pm2 start src/bot/core/BotClient.js --name "bleed-bot"
      ```
    </Note>
  </Accordion>
  
  <Accordion icon="shield-check" title="Production considerations">
    - Use environment variables for sensitive data
    - Set up proper logging
    - Implement error tracking
    - Configure automatic restarts
    - Set up regular backups for database
  </Accordion>
</AccordionGroup>

## Advanced Configuration

<AccordionGroup>
  <Accordion icon="plug" title="API Server">
    The built-in API server allows remote management:
    - Enable in `config.json`
    - Configure port and secret key
    - Access endpoints like `/api/commands`
  </Accordion>
  
  <Accordion icon="database" title="Database Setup">
    MongoDB integration provides:
    - Persistent data storage
    - User profiles
    - Server settings
    - Command statistics
    
    Enable in config and provide connection URI
  </Accordion>
  
  <Accordion icon="puzzle-piece" title="Custom Commands">
    Easily add new commands:
    1. Create new file in `src/commands/[category]`
    2. Follow the command template
    3. The bot will auto-load your command
  </Accordion>
</AccordionGroup>
